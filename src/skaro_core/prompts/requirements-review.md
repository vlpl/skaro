Analyze ALL requirements for quality issues.

Check for:
## 1. Duplicates
- Requirements that say the same thing differently
- Overlapping requirements that should be merged

## 2. Contradictions
- Requirements that conflict with each other
- Incompatible constraints

## 3. Incompleteness
- Missing acceptance criteria
- Vague or ambiguous language
- Undefined terms or metrics
- Missing edge cases

## 4. Priorities & Dependencies
- Requirements that depend on each other
- Missing prerequisite requirements
- Circular dependencies

## 5. Quality Issues
- Non-testable requirements
- Requirements that mix multiple concerns
- Inconsistent naming or terminology

## 6. Recommendations
- Suggested merges
- Suggested splits
- Suggested clarifications
- Suggested additions

Be specific — reference requirement IDs when flagging issues.

Return ONLY the review report. No preamble. No markdown fences.
